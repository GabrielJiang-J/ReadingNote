### 什么是代码审计
	代码审计是对一个软件的源码的综合分析。在评估、验证代码的成熟度和可维护性的时候，代码审计是很重要的安全处理过程，并且代码审计可以确保软件产品的无缝易手。

### 为什么要做代码审计
	代码审计包含下列基本目标：
		1. 熟悉、了解当前项目的结构和功能；
		2. 寻找、发现已存在和潜在的bug；
		3. 寻找、发现安全缺陷和漏洞；
		4. 验证性能和可扩展性；
		5. 评估代码的可维护性程度，以及相关的风险和成本；
		6. 验证与相关的开发标准、准则和最佳实践的一致程度；

### 什么是高效的代码审计方式
	代码审计分为两种方式：人工评审和自动评审；
	比较平衡的的代码审计策略是并用人工评审和自动评审。自动评审是采用自动化工具，对常见问题和漏洞进行搜索。人工评审可以检测出更复杂和更细微的问题。
	Manual analysis should be performed per major solution component and then the whole solution, starting with high-risk checklist issues and working all the way down to low-risk ones.

### 如何进行代码审计
	1. 基础的代码学习，以熟悉当前软件项目；
	2. 使用自动化工具对常见问题、漏洞和违规进行检测排查；
	3. 进行人工分析，来发现bug、性能瓶颈、安全漏洞和可维护性风险；
	4. 结合自动分析和人工分析的结果，形成全面的报告，并提供总结、发现问题的列表以及建议；

### 高效代码审计小贴士
	1. 定义代码审计的范围，建立代码审计的checklist，确保团队成员的理解一致性以及能够保证关键问题能够被处理和解决；
	2. 同时使用人工和自动评审，达到高效的评审；
	3. 避免“问责游戏”。不要在每次发现问题是就对研发人员追责，而是应该建立积极的安全文化，促进团队的学习和成长；
	4. 考虑引进第三方来做代码审计。这样能帮助团队发现在当前团队中认为不是问题的问题；
	5. 应该执行日常的代码审计活动；
